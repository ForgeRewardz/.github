# Rewardz Protocol 🎁

Hey Solana devs, if you're tired of designing rewards in the dark — guessing who your users are, hoping the campaign hits the right wallets, building bespoke indexers and claim flows from scratch every time — we've got you. We're the devs behind **Rewardz**: a shared rewards layer for Solana, sitting on top of an open-source program analytics tool called **Solscope**. Know your users first, then incentivize them right. Built for Colosseum Frontier, shipping live. ⚡

## Why Rewardz? (Because Reward Programs Shouldn't Take 6 Weeks 😩)

- **Solscope: Real Analytics, Not Vibes 📊** — Drop in a program ID, upload an IDL, see exactly who's calling what. Unique wallets per instruction, frequency, active windows, retention curves. Devnet AND mainnet. Free, open source, MIT.
- **Quick, Easy Onboarding 🏎️** — Pick curated DeFi actions (Jupiter, Kamino, Meteora, Jito, etc.) or upload your own IDL. Set rewards per action. Fund escrow. Live. That's it.
- **Onchain or Points, Your Call 🪙** — Pay in any SPL token via a Pinocchio escrow with trustless claim attestations, or run pre-token campaigns with offchain points and convert to airdrop later. Or both, in one rule.
- **Composable Incentives 🧱** — Protocol A rewards Jupiter swaps *into* their token, Protocol B rewards swaps *out of* theirs. Same user action triggers both. Incentives stack instead of compete.
- **Shared Infrastructure, Not Yours to Build 🛡️** — Indexing, attribution, attestations, claim flows — we run it once for everyone. Your team focuses on the product, not the plumbing.

We're targeting Solana builders running campaigns, launching tokens, retaining users, or just trying to figure out who's actually using their program. Integrate Rewardz, ship a real reward program in minutes, stop reinventing wheels. 👀
